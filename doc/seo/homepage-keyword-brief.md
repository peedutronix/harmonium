# Homepage Keyword Brief

## Page

- Primary page: `/`
- Supporting tool page: `/keyboard`

## Primary goal

Win early traction on a narrow but relevant keyword cluster around online harmonium usage, then expand into supporting guides and practice-oriented pages.

## Primary keywords

- `web harmonium`
- `online harmonium`
- `play harmonium online`

## Secondary keywords

- `virtual harmonium`
- `harmonium keyboard`
- `harmonium keyboard notes`
- `sargam notes harmonium`
- `practice harmonium online`

## Search intent

This keyword cluster is mostly tool-intent plus light informational intent.

Users typically want:

- something playable immediately
- no download friction
- keyboard or touch support
- note labels or practice hints

They usually do not want:

- a complex SaaS dashboard
- account setup before first use
- abstract marketing copy without a usable tool

## Recommended page targeting

- `/`
  target broad “what is this / can I use it now” intent
- `/keyboard`
  target “open the tool and play now” intent
- `/blog/how-to-play-harmonium-online-for-free`
  target beginner informational intent
- `/blog/harmonium-keyboard-notes-for-beginners`
  target note mapping and learning intent

## Homepage angle

Position Play Harmonium as:

`A fast, browser-based harmonium practice tool with keyboard shortcuts, touch controls, Sargam labels, and no download requirement.`

## Title and description direction

Homepage title direction:

- `Play Harmonium Online | Play Harmonium`

Homepage description direction:

- `Practice harmonium in your browser with touch, keyboard shortcuts, Sargam labels, octave controls, and transpose.`

Keyboard page title direction:

- `Play Harmonium Keyboard | Practice Mode`

## Content requirements for `/`

- Strong H1 with the main keyword cluster implied naturally
- Clear supporting paragraph that confirms browser-based usage
- Visible CTA to `/keyboard`
- Preview of the play surface
- Feature copy around practice value
- FAQ answering common friction questions
- Links to blog guides

## Internal linking plan

Homepage should link to:

- `/keyboard`
- `/blog`
- beginner guide article
- keyboard notes article
- pricing page only as a secondary path

Blog posts should link back to:

- `/`
- `/keyboard`

Keyboard page should link to:

- `/`
- `/blog`

## Risks

- Over-optimizing copy around the keyword can make the page feel spammy.
- If `/keyboard` has no distinct purpose, it can dilute the homepage instead of supporting it.
- If the homepage hides too much real text behind UI chrome, indexing quality may suffer.

## Near-term measurement

Use Search Console and analytics to watch:

- impressions for `web harmonium` variants
- clicks to `/`
- clicks from `/` to `/keyboard`
- engagement on `/keyboard`
- blog entry page impressions for long-tail queries

## Next SEO tasks

- Create a dedicated internal link from homepage hero to `/keyboard`
- Keep metadata clean and unique across `/` and `/keyboard`
- Add more supporting guides only after the first two posts stabilize
- Review whether `/keyboard` needs a richer text block for search support
